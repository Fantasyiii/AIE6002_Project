# VibeMatch 技术栈

## 项目概述
VibeMatch 是一个基于 RAG（Retrieval-Augmented Generation）的语义化电影推荐引擎，用户可通过自然语言描述"感觉"来搜索电影，系统返回精准推荐并解释匹配原因。

---

## 核心技术栈

| 层级 | 技术选型 | 用途说明 |
|:---|:---|:---|
| **编程语言** | Python 3.10+ | 主开发语言 |
| **RAG 框架** | LangChain | 编排 RAG 流程：文档加载、文本分块、检索链 |
| **向量数据库** | ChromaDB | 本地存储电影剧情的向量索引，支持余弦相似度检索 |
| **Embedding 模型** | HuggingFace `all-MiniLM-L6-v2` | 本地运行，免 API 费用，将文本转换为 384 维向量 |
| **LLM** | GPT-4o-mini | 根据检索到的上下文生成最终推荐 + 解释 |
| **Web UI** | Streamlit | 构建交互式聊天界面，支持流式输出和溯源面板 |

---

## 详细组件与依赖

### 1. 数据层
| 组件 | 技术 | 说明 |
|:---|:---|:---|
| 数据集 | TMDB 5000 Movie Dataset (Kaggle) | 包含电影标题、类型、年份、剧情概述 |
| 数据处理 | Pandas | 数据清洗、缺失值处理 |
| 文本分块 | `RecursiveCharacterTextSplitter` (LangChain) | chunk_size=500, overlap=50 |

### 2. 向量化与检索层
| 组件 | 技术 | 说明 |
|:---|:---|:---|
| Embedding 模型 | `HuggingFaceEmbeddings` (`all-MiniLM-L6-v2`) | 本地模型，免 API 费用 |
| 向量存储 | ChromaDB | 本地持久化，自动保存到磁盘 |
| 检索策略 | Cosine Similarity + MMR | 平衡相关性与多样性 |
| 检索参数 | Top-K = 5 | 返回最相关的 5 个候选电影 |

### 3. 生成层
| 组件 | 技术 | 说明 |
|:---|:---|:---|
| LLM | `ChatOpenAI` (GPT-4o-mini) | 通过 LangChain 统一调用 |
| Prompt 模板 | `ChatPromptTemplate` (LangChain) | 将检索到的上下文 + 用户问题组装成结构化提示 |
| 输出解析器 | `StrOutputParser` | 解析 LLM 返回的文本 |

### 4. 应用层
| 组件 | 技术 | 说明 |
|:---|:---|:---|
| Web 框架 | Streamlit | 快速原型，支持聊天界面和流式输出 |
| 会话管理 | `st.session_state` | 维护多轮对话历史 |
| 溯源展示 | `st.expander` | 展示检索到的源数据（电影标题+剧情） |

## 技术选型理由

1. **ChromaDB 替代 FAISS**: 内置持久化和 metadata 过滤，4803 部电影规模下完全够用，且与 LangChain 集成更简洁
2. **all-MiniLM-L6-v2 作为默认 Embedding**: 本地运行零费用，384 维向量存储高效，语义相似度任务表现足够
3. **GPT-4o-mini 作为默认 LLM**: 性价比高、响应快、API 稳定，适合学术项目实验
4. **版本号使用 `~=`**: 锁定主版本和次版本，确保项目可稳定复现
